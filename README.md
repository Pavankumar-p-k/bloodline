### 📦 Installation

**Prerequisites:** Node.js >= 18, npm or yarn

```bash
# Clone
git clone https://github.com/Pavankumar-p-k/bloodline.git
cd bloodline

# Install backend dependencies
cd backend && npm install && cd ..

# Install frontend dependencies
cd frontend && npm install && cd ..
```

### 🏃 Run with Local Auth (no Supabase needed)

```bash
# Terminal 1: Backend
cd backend && node local-dev.js

# Terminal 2: Frontend
cd frontend && npm run dev
```

Open **http://localhost:3000** — log in with any test account below.

### ☁️ Run with Supabase

```bash
cp frontend/.env.example frontend/.env.local
cp backend/.env.example backend/.env
# Fill in your Supabase credentials, then:
cd backend && npm start     # Terminal 1
cd frontend && npm run dev  # Terminal 2
```

## 🧰 Local Development

### Without Supabase (recommended)

```bash
cd backend && node local-dev.js
```

The `local-dev.js` server handles all auth and API in memory — seeds 3 test accounts automatically. The frontend auto-detects missing Supabase env vars and falls back to localStorage-based auth.

### With Supabase

1. Create a project at [supabase.com](https://supabase.com)
2. Run `db/supabase_setup.sql` in the SQL editor
3. Copy project URL & anon key to `frontend/.env.local`
4. Copy project URL & service key to `backend/.env`
5. Set `SEED_DEFAULT_USERS=true` in `backend/.env`
